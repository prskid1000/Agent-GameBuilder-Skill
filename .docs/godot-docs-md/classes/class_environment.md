## Environment

**Inherits:** [Resource](class_resource.md#class_Resource) **<** [RefCounted](class_refcounted.md#class_RefCounted) **<** [Object](class_object.md#class_Object)

Resource for environment nodes (like [WorldEnvironment](class_worldenvironment.md#class_WorldEnvironment)) that define multiple rendering options.

### Description

Resource for environment nodes (like [WorldEnvironment](class_worldenvironment.md#class_WorldEnvironment)) that define multiple environment operations (such as background [Sky](class_sky.md#class_Sky) or [Color](class_color.md#class_Color), ambient light, fog, depth-of-field...). These parameters affect the final render of the scene. The order of these operations is:

- Depth of Field Blur

- Auto Exposure

- Glow

- Tonemap

- Adjustments

<span id="enum_Environment_ToneMapper"></span>

enum **ToneMapper**: [ðŸ”—](class_environment.md#enum_Environment_ToneMapper)

<span id="class_Environment_constant_TONE_MAPPER_LINEAR"></span>

[ToneMapper](class_environment.md#enum_Environment_ToneMapper) **TONE_MAPPER_LINEAR** = `0`

Does not modify color data, resulting in a linear tonemapping curve which unnaturally clips bright values, causing bright lighting to look blown out. The simplest and fastest tonemapper.

<span id="class_Environment_constant_TONE_MAPPER_REINHARDT"></span>

[ToneMapper](class_environment.md#enum_Environment_ToneMapper) **TONE_MAPPER_REINHARDT** = `1`

A simple tonemapping curve that rolls off bright values to prevent clipping. This results in an image that can appear dull and low contrast. Slower than [TONE_MAPPER_LINEAR](class_environment.md#class_Environment_constant_TONE_MAPPER_LINEAR).

 **Note:** When [tonemap_white](class_environment.md#class_Environment_property_tonemap_white) is left at the default value of `1.0`, [TONE_MAPPER_REINHARDT](class_environment.md#class_Environment_constant_TONE_MAPPER_REINHARDT) produces an identical image to [TONE_MAPPER_LINEAR](class_environment.md#class_Environment_constant_TONE_MAPPER_LINEAR).

<span id="class_Environment_constant_TONE_MAPPER_FILMIC"></span>

[ToneMapper](class_environment.md#enum_Environment_ToneMapper) **TONE_MAPPER_FILMIC** = `2`

Uses a film-like tonemapping curve to prevent clipping of bright values and provide better contrast than [TONE_MAPPER_REINHARDT](class_environment.md#class_Environment_constant_TONE_MAPPER_REINHARDT). Slightly slower than [TONE_MAPPER_REINHARDT](class_environment.md#class_Environment_constant_TONE_MAPPER_REINHARDT).

 **Note:** This tonemapper does not support HDR output because it produces output in the SDR range. It is recommended to use a different tonemapper when rendering to an HDR screen.

<span id="class_Environment_constant_TONE_MAPPER_ACES"></span>

[ToneMapper](class_environment.md#enum_Environment_ToneMapper) **TONE_MAPPER_ACES** = `3`

Uses a high-contrast film-like tonemapping curve and desaturates bright values for a more realistic appearance. Slightly slower than [TONE_MAPPER_FILMIC](class_environment.md#class_Environment_constant_TONE_MAPPER_FILMIC).

 **Note:** This tonemapping operator is called "ACES Fitted" in Godot 3.x.

 **Note:** This tonemapper does not support HDR output because it produces output in the SDR range. It is recommended to use a different tonemapper when rendering to an HDR screen.

<span id="class_Environment_constant_TONE_MAPPER_AGX"></span>

[ToneMapper](class_environment.md#enum_Environment_ToneMapper) **TONE_MAPPER_AGX** = `4`

Uses an adjustable film-like tonemapping curve and desaturates bright values for a more realistic appearance. Better than other tonemappers at maintaining the hue of colors as they become brighter. The slowest tonemapping option.

<span id="enum_Environment_GlowBlendMode"></span>

enum **GlowBlendMode**: [ðŸ”—](class_environment.md#enum_Environment_GlowBlendMode)

<span id="class_Environment_constant_GLOW_BLEND_MODE_ADDITIVE"></span>

[GlowBlendMode](class_environment.md#enum_Environment_GlowBlendMode) **GLOW_BLEND_MODE_ADDITIVE** = `0`

Adds the glow effect to the scene.

<span id="class_Environment_constant_GLOW_BLEND_MODE_SCREEN"></span>

[GlowBlendMode](class_environment.md#enum_Environment_GlowBlendMode) **GLOW_BLEND_MODE_SCREEN** = `1`

Adds the glow effect to the scene after modifying the glow influence based on the scene value; dark values will be highly influenced by glow and bright values will not be influenced by glow. This approach avoids bright values becoming overly bright from the glow effect. [tonemap_white](class_environment.md#class_Environment_property_tonemap_white) is used to determine the maximum scene value where the glow should have no influence. When [tonemap_mode](class_environment.md#class_Environment_property_tonemap_mode) is set to [TONE_MAPPER_LINEAR](class_environment.md#class_Environment_constant_TONE_MAPPER_LINEAR) and [Viewport.use_hdr_2d](class_viewport.md#class_Viewport_property_use_hdr_2d) is `true`, the parent window's [Window.get_output_max_linear_value()](class_window.md#class_Window_method_get_output_max_linear_value) will be used as the maximum scene value.

<span id="class_Environment_constant_GLOW_BLEND_MODE_SOFTLIGHT"></span>

[GlowBlendMode](class_environment.md#enum_Environment_GlowBlendMode) **GLOW_BLEND_MODE_SOFTLIGHT** = `2`

Adds the glow effect to the tonemapped image after modifying the glow influence based on the image value; dark values and bright values will not be influenced by glow and mid-range values will be highly influenced by glow. This approach avoids bright values becoming overly bright from the glow effect. The glow will have the largest influence on image values of `0.25` and will have no influence when applied to image values greater than `1.0`.

 **Note:** This blend mode does not support HDR output because expects a maximum output value of `1.0`. It is recommended to use a different blend mode when rendering to an HDR screen.

Replaces all pixels' color by the glow effect. This can be used to simulate a full-screen blur effect by tweaking the glow parameters to match the original image's brightness or to preview glow configuration in the editor.

<span id="class_Environment_property_fog_aerial_perspective"></span>

[float](class_float.md#class_float) **fog_aerial_perspective** = `0.0` [ðŸ”—](class_environment.md#class_Environment_property_fog_aerial_perspective)

- void **set_fog_aerial_perspective** ( value: [float](class_float.md#class_float) )
- [float](class_float.md#class_float) **get_fog_aerial_perspective** ( )

If set above `0.0` (exclusive), blends between the fog's color and the color of the background [Sky](class_sky.md#class_Sky), as read from the radiance cubemap. This has a small performance cost when set above `0.0`. Must have [background_mode](class_environment.md#class_Environment_property_background_mode) set to [BG_SKY](class_environment.md#class_Environment_constant_BG_SKY).

This is useful to simulate aerial perspective in large scenes with low density fog. However, it is not very useful for high-density fog, as the sky will shine through. When set to `1.0`, the fog color comes completely from the [Sky](class_sky.md#class_Sky). If set to `0.0`, aerial perspective is disabled.

Notice that this does not sample the [Sky](class_sky.md#class_Sky) directly, but rather the radiance cubemap. The cubemap is sampled at a mipmap level depending on the depth of the rendered pixel; the farther away, the higher the resolution of the sampled mipmap. This results in the actual color being a blurred version of the sky, with more blur closer to the camera. The highest mipmap resolution is used at a depth of [Camera3D.far](class_camera3d.md#class_Camera3D_property_far).

- [bool](class_bool.md#class_bool) **is_volumetric_fog_temporal_reprojection_enabled** ( )

Enables temporal reprojection in the volumetric fog. Temporal reprojection blends the current frame's volumetric fog with the last frame's volumetric fog to smooth out jagged edges. The performance cost is minimal; however, it leads to moving [FogVolume](class_fogvolume.md#class_FogVolume) s and [Light3D](class_light3d.md#class_Light3D) s "ghosting" and leaving a trail behind them. When temporal reprojection is enabled, try to avoid moving [FogVolume](class_fogvolume.md#class_FogVolume) s or [Light3D](class_light3d.md#class_Light3D) s too fast. Short-lived dynamic lighting effects should have [Light3D.light_volumetric_fog_energy](class_light3d.md#class_Light3D_property_light_volumetric_fog_energy) set to `0.0` to avoid ghosting.
